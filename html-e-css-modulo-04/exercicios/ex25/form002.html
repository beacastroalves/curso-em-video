Meu primeiro formulário - Adicionando autocomplete e label



  • autocomplete="off" é colocado dentro da tag form e é designado para não deixar aparecer sugestões para o usuário digitar na caixa. É totalmente opção e em muitos casos pode facilitar a vida do usuário. Depende do objetivo.

  • Label é como uma etiqueta que você coloca em um campo de formulário para dizer ao usuário o que ele deve preencher ou selecionar. A utilidade dela é criar uma relação entre esse rótulo (titulo) e o input que é a caixa com o conteudo ou marcação. Ajuda também a expandir a area sensivel ao toque, e se o usuário clica nesse titulo/etiqueta o for="" da label vai acionar o intput com o id="" relacionado. (Eles devem ser iguais)
    • A correspondência entre o for do <label> e o id do elemento de formulário é fundamental para a acessibilidade e usabilidade. é uma convenção e um padrão comum em muitos trabalhos profissionais, mas não é uma regra estrita.
    • Usar um prefixo como "i" seguido do name no id é uma convenção profissional comum para:
      • Melhorar a organização
      • Evitar conflitos de id
      • Facilitar o desenvolvimento com JavaScript e CSS
      Mas não é uma exigência da especificação HTML. A consistência dentro de um projeto é mais importante do que seguir um prefixo específico. Você pode encontrar outros padrões de nomenclatura em diferentes projetos.


Código do formulário acima

    
      <form autocomplete="off">
        <p>
          <label for="inome" >Nome:</label>
          <input type="text" name="nome" id="inome">
        </p>
        <p>
          <label for="isobrenome">Sobrenome:</label>
          <input type="text" name="sobrenome" id="isobrenome">
        </p>
        <p><input type="submit" value="Enviar"></p>
      </form>
