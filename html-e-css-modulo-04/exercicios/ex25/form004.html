Teste de formulário - Criando caixas de texto e de senha



Código do formulário acima

    
       <form action="cadastro.php" method="post" autocomplete="on">
        <p>
          <label for="iusu">Usuário:</label>
          <input type="text" name="usu" id="iusu" required minlength="5" maxlength="15" size="15" placeholder="nome do usuário" autocomplete="username">
        </p>
        <p>
          <label for="isen">Senha:</label>
          <input type="password" name="sen" id="isen" minlength="8" maxlength="20" size="15" placeholder="minimo 8 caracteres" autocomplete="current-password">
        </p>
        <p>
          <input type="submit" value="Enviar">
          <input type="reset" value="Limpar">
        </p>
      </form>

  • Atributos da Tag <form>:
    • action="cadastro.php"
      • Significado: Especifica o URL para onde os dados do formulário serão enviados quando o formulário for submetido.
      • Para que serve: Define o script ou página no servidor que irá processar os dados inseridos pelo usuário no formulário.
    • method="post"
      • Significado: Define o método HTTP usado para enviar os dados do formulário.
      • Para que serve: Indica como os dados do formulário serão enviados ao servidor. O método POST envia os dados no corpo da requisição HTTP, sendo geralmente mais adequado para dados sensíveis ou grandes quantidades de informação, pois não os expõe na URL.
    • autocomplete="on"
      • Significado: Especifica se o navegador deve tentar preencher automaticamente os campos do formulário com base em entradas anteriores do usuário.
      • Para que serve: Melhora a experiência do usuário, sugerindo valores para os campos com base no histórico de preenchimento, agilizando o processo de preenchimento.

  • Atributos da Tag <label>:
    • for="iusu"
      • Significado: Associa este rótulo (o texto "Usuário:") ao elemento de formulário com o id correspondente ("iusu").
      • Para que serve: Melhora a acessibilidade, permitindo que leitores de tela anunciem o rótulo quando o elemento de formulário associado é selecionado. Além disso, clicar no rótulo foca o elemento de formulário.
    • for="isen"
      • Significado: Associa este rótulo (o texto "Senha:") ao elemento de formulário com o id correspondente ("isen").
      • Para que serve: Similar ao for="iusu", melhora a acessibilidade e usabilidade ao associar o rótulo ao campo de senha.

  • Atributos da Tag <input type="text">:
    • type="text"
      • Significado: Define o tipo do elemento de entrada como um campo de texto de linha única.
      • Para que serve: Permite que o usuário digite texto livremente.
    • name="usu"
      • Significado: Define o nome do elemento de entrada. Este nome é usado para identificar os dados deste campo quando o formulário é enviado ao servidor.
      • Para que serve: Permite que o servidor acesse o valor digitado pelo usuário neste campo através do nome "usu".
    • id="iusu"
      • Significado: Define um identificador único para este elemento dentro do documento HTML.
      • Para que serve: Permite que o elemento seja referenciado por outros elementos (como a tag <label> através do atributo for), por JavaScript para manipulação dinâmica e por CSS para aplicar estilos específicos.
    • required
      • Significado: Especifica que este campo deve ser preenchido antes que o formulário possa ser enviado.
      • Para que serve: Garante que o usuário forneça um valor para este campo, impedindo o envio do formulário se estiver vazio.
    • minlength="5"
      • Significado: Especifica o número mínimo de caracteres exigidos para o valor deste campo.
      • Para que serve: Impõe uma validação no lado do cliente, garantindo que o usuário digite pelo menos 5 caracteres antes de poder enviar o formulário.
    • maxlength="15"
      • Significado: Especifica o número máximo de caracteres permitidos para o valor deste campo.
      • Para que serve: Impõe um limite no lado do cliente para o número de caracteres que o usuário pode digitar neste campo.
    • size="15"
      • Significado: Define a largura visível do campo de entrada (em caracteres).
      • Para que serve: Controla o tamanho visual do campo de texto na tela, embora não limite o número de caracteres que podem ser digitados (isso é feito por maxlength).
    • placeholder="nome do usuário"
      • Significado: Fornece uma breve dica ou descrição dentro do campo de entrada antes que o usuário digite algo. O texto do placeholder desaparece quando o usuário foca e começa a digitar no campo.
      • Para que serve: Oferece uma instrução ou exemplo do tipo de informação esperada no campo.
    • autocomplete="username"
      • Significado: Fornece uma dica ao navegador sobre o tipo de informação esperada neste campo para fins de preenchimento automático. O valor "username" indica que se espera um nome de usuário.
      • Para que serve: Permite que o navegador ofereça sugestões de nomes de usuário previamente armazenados, melhorando a conveniência para o usuário.

  • Atributos da Tag <input type="password">:
    • type="password"
      • Significado: Define o tipo do elemento de entrada como um campo de senha.
      • Para que serve: O texto digitado neste campo é geralmente mascarado (substituído por asteriscos ou bolinhas) para proteger a informação sensível.
    • name="sen"
      • Significado: Define o nome do elemento de entrada para a senha.
      • Para que serve: Permite que o servidor acesse a senha digitada pelo usuário através do nome "sen".
    • id="isen"
      • Significado: Define um identificador único para o campo de senha.
      • Para que serve: Permite que o elemento seja referenciado pela tag <label> (através do for) e por JavaScript/CSS.
    • required
      • Significado: Especifica que este campo deve ser preenchido antes que o formulário possa ser enviado.
      • Para que serve: Garante que o usuário forneça uma senha, impedindo o envio do formulário se estiver vazia.
    • minlength="8"
      • Significado: Especifica o número mínimo de caracteres exigidos para a senha.
      • Para que serve: Garante que a senha tenha pelo menos 8 caracteres.
    • maxlength="20"
      • Significado: Especifica o número máximo de caracteres permitidos para a senha.
      • Para que serve: Limita o tamanho máximo da senha.
    • size="15"
      • Significado: Define a largura visível do campo de senha.
      • Para que serve: Controla o tamanho visual do campo de senha.
    • placeholder="minimo 8 caracteres"
      • Significado: Fornece uma dica sobre o formato esperado para a senha.
      • Para que serve: Informa o usuário sobre a exigência de um mínimo de 8 caracteres.
    • autocomplete="current-password"
      • Significado: Fornece uma dica ao navegador sobre o tipo de informação esperada neste campo para fins de preenchimento automático. O valor "current-password" indica que se espera a senha atual do usuário.
      • Para que serve: Permite que o navegador ofereça sugestões de senhas previamente armazenadas para o contexto atual.

  • Atributos da Tag <input type="submit">:
    • type="submit"
      • Significado: Define o tipo do elemento de entrada como um botão de envio.
      • Para que serve: Ao ser clicado, este botão envia os dados do formulário para o URL especificado no atributo action da tag <form>.
    • value="Enviar"
      • Significado: Define o texto que será exibido no botão de envio.
      • Para que serve: Apresenta um texto claro para o usuário indicando a ação que o botão realizará.

  • Atributos da Tag <input type="reset">:
    • type="reset"
      • Significado: Define o tipo do elemento de entrada como um botão de reset.
      • Para que serve: Ao ser clicado, este botão redefine todos os campos do formulário para seus valores iniciais.
    • value="Limpar"
      • Significado: Define o texto que será exibido no botão de reset.
      • Para que serve: Apresenta um texto claro para o usuário indicando a ação de limpar o formulário.
