  • Metodo GET é uma maneira de solicitar informações de um servidor web. Cuidado com conteudos sensíveis. As principais características do método GET.
    • Solicita dados: Seu objetivo principal é recuperar dados do servidor.
    • Dados na URL: Os parâmetros da sua requisição (as informações adicionais que você está enviando) são geralmente anexados à URL. Por exemplo: www.exemplo.com/produtos?categoria=livros&preco_max=50.
    • Não altera dados: Uma requisição GET não deve ter o efeito de modificar os dados no servidor. É uma operação segura e idempotente (repetir a mesma requisição várias vezes deve ter o mesmo resultado).
    • Limite de dados: Devido à forma como os dados são enviados na URL, geralmente há um limite para a quantidade de informações que podem ser enviadas em uma requisição GET.
    • Pode ser cacheado (e não estamos falando de cabelos hihi): As respostas das requisições GET podem ser armazenadas em cache pelos navegadores e servidores, o que pode melhorar o desempenho.

  • Metodo POST é usado para enviar dados para um servidor web para serem processados. As principais características do método POST.
    • Envia dados: Seu objetivo principal é enviar dados para o servidor, frequentemente para criar ou atualizar algum recurso.
    • Dados no corpo da requisição: Os parâmetros da sua requisição são enviados no corpo da mensagem HTTP, não na URL. Isso torna os dados menos visíveis e permite enviar informações mais sensíveis de forma mais segura (embora ainda precise de HTTPS para criptografia).
    • Pode alterar dados: As requisições POST podem ter o efeito de modificar os dados no servidor (por exemplo, criar um novo usuário, adicionar um comentário, fazer um pedido). Não é uma operação necessariamente segura ou idempotente.
    • Sem limite de dados (prático): Teoricamente, não há um limite estrito para a quantidade de dados que podem ser enviados em uma requisição POST, embora servidores possam ter suas próprias configurações de limite.
    • Geralmente não é cacheado: As respostas das requisições POST geralmente não são armazenadas em cache, pois o resultado pode depender dos dados enviados.
